## Contributing

- **Opening Issues**: Feel free to open GitHub issues for bugs, feature requests, or questions. Please include a clear description, steps to reproduce (for bugs), and any relevant context.

- **Pull Requests**: Fork the repository, create a feature branch, make your changes, and submit a pull request. Ensure your code follows the project's style and includes appropriate tests.

- **Development Setup**: After forking and cloning, run `neva mod init` to initialize the module. Use `neva run main` to test your changes locally.
